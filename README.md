# Multi-Service Security System (Cyber Lab 2)

Проект представляет собой комплексную систему информационной безопасности, реализующую управление секретами (Vault) и сервис цифровой подписи. Система построена на микросервисной архитектуре с использованием современных методов криптографии.


## Архитектура проекта

Проект разделен на три ключевых компонента:

1.  **Backend Vault (Порт 8003)**: Управляет жизненным циклом мастер-ключа и хранением зашифрованных секретов.
2.  **Backend Signature (Порт 8001)**: Отвечает за вычисление хеш-сумм и формирование цифровых подписей файлов.
3.  **Frontend**: Веб-интерфейс для взаимодействия с сервисами без использования сторонних инструментов типа Postman.

---

## Основные функции

### 1. Схема разделения секрета (Shamir-like/XOR)
Для обеспечения высокой доступности и безопасности мастер-ключ не хранится в открытом виде. Он восстанавливается в оперативной памяти только после ввода **двух независимых частей ключа**.
*   **Статус "Sealed"**: Хранилище заблокировано, доступ к секретам невозможен.
*   **Статус "Unsealing"**: Получена первая часть ключа.
*   **Статус "Unsealed"**: Мастер-ключ восстановлен, функции шифрования активны.

### 2. Хранилище секретов
*   **Алгоритм**: Fernet (симметричное шифрование на базе AES-128 в режиме CBC с HMAC SHA256).
*   **Хранение**: Файл `vault_storage.json`, данные в котором защищены мастер-ключом.

### 3. Цифровая подпись
*   **Алгоритм**: RSA-PSS (Probabilistic Signature Scheme).
*   **Целостность**: Использование надежных хеш-функций для подтверждения неизменности файлов.
